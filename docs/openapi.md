# OpenAPI Documentation

UNTOLD publishes machine-readable API specifications for the main REST API and the external API Gateway.

## Specifications

| Spec | URL (development) | Format |
|------|-------------------|--------|
| **Main API** | `http://localhost:8000/openapi.json` | OpenAPI 3.x (auto-generated by FastAPI) |
| **Gateway API** | `http://localhost:8000/gateway/openapi.json` | OpenAPI 3.1 (static spec) |

## Interactive explorers

| UI | URL | Availability |
|----|-----|--------------|
| Swagger UI | `/docs` | Non-production only |
| ReDoc | `/redoc` | Non-production only |
| Gateway Swagger | `/gateway/docs` | All environments (configure access in prod) |

Production disables `/docs` and `/redoc` on the main API (`settings.is_production`).

## Export OpenAPI spec

### Script (recommended)

```bash
cd backend
python scripts/export_openapi.py
# Writes: docs/openapi/untold-api.json
```

### Manual curl

```bash
curl -s http://localhost:8000/openapi.json | jq . > docs/openapi/untold-api.json
curl -s http://localhost:8000/gateway/openapi.json | jq . > docs/openapi/untold-gateway.json
```

### From Python

```python
from app.main import app
import json
print(json.dumps(app.openapi(), indent=2))
```

## CI integration

Add to release pipeline:

```yaml
- name: Export OpenAPI
  run: |
    cd backend
    pip install -r requirements.txt
    python scripts/export_openapi.py
- name: Upload artifact
  uses: actions/upload-artifact@v4
  with:
    name: openapi
    path: docs/openapi/
```

Commit exported specs on version tags for integrator stability.

## Client generation

Generate TypeScript or Python clients from exported spec:

```bash
# TypeScript (openapi-generator)
npx @openapitools/openapi-generator-cli generate \
  -i docs/openapi/untold-api.json \
  -g typescript-axios \
  -o clients/typescript

# Python
openapi-generator generate \
  -i docs/openapi/untold-api.json \
  -g python \
  -o clients/python
```

## Documenting new endpoints

FastAPI auto-documents from:

1. **Route decorator** — `summary`, `description`, `response_model`
2. **Pydantic schemas** — field descriptions via `Field(description="...")`
3. **Tags** — `APIRouter(tags=["Studio"])`
4. **Status codes** — `responses={404: {"description": "Not found"}}`

Example:

```python
@router.post(
    "/generate",
    response_model=GenerationResponse,
    summary="Generate AI image",
    description="Enqueue an image generation job for the project.",
)
def generate_image(...):
    ...
```

## Gateway spec maintenance

Gateway OpenAPI is maintained manually in:

```
backend/app/gateway/openapi_spec.py
```

Update when adding gateway routes in `app/gateway/router.py`.

## Versioning

| Field | Source |
|-------|--------|
| `info.version` | `settings.app_version` (main API) |
| Gateway version | `OPENAPI_SPEC["info"]["version"]` |

Breaking changes require bumping API prefix (`/api/v2`) or gateway version header.

## Authentication in Swagger

Development `/docs`:

1. Click **Authorize**
2. Enter `Bearer <access_token>` from `POST /api/v1/auth/login`

Gateway `/gateway/docs`:

1. Enter API key in `X-API-Key` header field
